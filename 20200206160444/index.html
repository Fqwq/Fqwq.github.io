oracle随记

2020-02-06 / 4 min read

oracle简述

oracle是一个关系型数据库

关系型数据库：指采用了关系模型来组织数据的数据库。
关系模型指的就是二维表格模型，而一个关系型数据库就是由二维表及其之间的联系所组成的一个数据组织。

oracle登录sql plus并管理用户信息

sys(系统管理员 最高权限)

system sysman(本地管理员 次高权限)

scott(普通用户 密码默认为tiger 默认未解锁)

登录语句

​ username/password [@server] [AS SYSDBA|SYSOPER]

查看登录用户

​ SHOW USER 查看当前用户

​ desc dba_users 通过数据字典查看其他用户

​ select username from dba_users 选取并查看在线用户

解锁或者锁定用户

​ alter user username account LOCK|UNLOCK 解锁或锁定用户

表空间

表空间 概述

表空间是数据库的逻辑数据存储空间

是数据库中开辟的一个个空间用于存储数据库的对象

数据库可以由多个表空间构成

使用describe/desc查看表空间

表空间的分类

永久表空间 数据库中永久化的对象(视图等)

临时表空间 数据库中执行过程的东西(执行操作后就删除)

UNDO表空间 数据库被修改之前的数据(撤销操作的存储地址)

管理用户表空间

查看用户的表空间

dba_tablespaces 管理员登陆后查看的数据字典

user_tablespaces 用户登录后查看的数据字典

​ desc dba_tablespaces 查看管理员的数据字典

tablespace_name 表空间名字

​ select tablespace_name from dba_tablespaces 查看管理员表空间

system 主要存储sys用户的表视图 系统表空间

sysaux example示例 oracle示例使用的表空间

undotbs1 存储撤销信息的表空间

temp 存储sql处理的表 临时表空间

users 主要存储数据库用户创建的对象

设置用户的默认或临时表空间

ALTER USER username DEFAULT|TEMPORARY TABLESPACE tablespace_name

创建表空间

CREATE [TEMPORARY] TABLESPACE tablespace_name TEMPFILE|DATAFILE 'xx.dbf' SIZE xx

创建永久或临时表空间

修改表空间

设置表空间联机/脱机状态

ALTER TABLESPACE tablespace_name ONLINE|OFFLINE

(数据字典中status字段为联机脱机状态)

增加与删除数据文件

ALTER TABLESPACE tablespace_name ADD DATAFILE 'xx.dbf' SIZE xx; 增加数据文件

ALTER TABLESPACE tablespace_name DROP DATAFILE 'filename.dbf'; 删除数据文件 不能删除表空间第一个数据文件

表

表 概述

表是表空间的基本储存单位

表是一个二维结构 由行和列构成

约定

每一列数据都必须具有相同的数据类型

列名唯一

每一行数据的唯一性

表中的数据类型

数值型

NUMBER(p,s) p有效位数 s小数点后位数

FLOAT(n) 存储二进制数据 转换时需要在数据*0.30103

**字符型 **

CHAR(n) 最大值为2000

NCHAR(n) 最大值为1000

VARCHAR(n) 最大值为4000

NVARCHAR(n) 最大值为4000

日期型

DATE

TMIESTAMP 时间戳类型

其他类型

BLOB 存放4gb的数据 以2进制储存的数据

CLOB 存放4gb的数据 以字符串储存的数据

创建表

CREATE TABLE table_name
(	
    column_name datatype , ...
	);

CREATE TABLE userinfo
(	id number(6,0),
    username varchar2(20),
	userpwd varchar2(20),
 	email varchar2(30),
	regdate date	);

修改表

在表中增加字段
ALTER TABLE table_name ADD column_name datatype;

ALTER TABLE userinfo ADD remarks varchar2(500);
修改表字段数据类型
ALTER TABLE table_name MODIFY column_name datatype;

ALTER TABLE userinfo MODIFY remarks varchar2(400);

修改时没有数据才能够实现

在表中删除字段
ALTER TABLE table_name DROP COLUMN column_name;

ALTER TABLE userinfo DROP COLUMN remarks;
在表中修改字段名
ALTER TABLE table_name RENAME COLUMN column_name TO new_column_name;

ALTER TABLE userinfo RENAME COLUMN email TO new_email;
